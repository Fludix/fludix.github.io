Privacy Policy

Last updated: July 19, 2026

This Privacy Policy explains what information CypherHound processes when you use it, and what happens to it.

1. What we process when you run a command

To execute a slash command, CypherHound temporarily handles:

  • The command name and the input you provide (e.g. a domain, IP address, CVE ID, or username);
  • Your Discord user ID and the server (guild) ID, as provided automatically by Discord with every interaction — this is standard for any Discord bot and is not something CypherHound can opt out of receiving.
As currently built, CypherHound does not write command inputs, results, or user/server IDs to any database or persistent log. Each request is processed in memory and discarded once a response is sent. If that changes — for example, if logging is added for abuse prevention or rate limiting — this policy will be updated to describe what's retained and for how long before that change ships.

2. Where your input goes

Depending on the command, your input is forwarded to the relevant public data source to retrieve results:

  • /whois — the domain is sent to the relevant public WHOIS registry server;
  • /geoip — the IP address is sent to ip-api.com;
  • /certs — the domain is sent to crt.sh (public certificate transparency search);
  • /cve — the CVE ID is sent to the National Vulnerability Database (nvd.nist.gov);
  • /username — the handle is checked against each listed platform's public profile URL to determine whether a page exists (no profile content is retrieved or stored).

These are independent third-party services with their own privacy practices, which we don't control. We only send the minimum needed (the query itself) — no additional Discord identifiers are forwarded to them.

3. What we don't do

  • We do not sell, rent, or share your data with advertisers or data brokers.
  • We do not build profiles of users or servers based on command history.
  • We do not access message content outside of the slash command inputs you explicitly provide.

4. Data retention

Because command inputs and results are not persistently stored (see Section 1), there is generally nothing to retain or delete. Standard Discord platform logs (e.g. gateway connection logs) may exist transiently at the hosting level for operational stability, and are not used for any purpose beyond keeping the bot running.

5. Children's privacy

CypherHound is not directed at children and does not knowingly process data from users below Discord's own minimum age requirement (13, or higher where required by local law).

6. Your choices

You can stop CypherHound from processing any data associated with you at any time by not using its commands, or by having a server administrator remove the bot from a server.

7. Changes to this policy

If our data practices change — most notably, if we begin retaining logs — this page will be updated and the "Last updated" date above will change accordingly.
